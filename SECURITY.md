# 安全策略

## 支持的版本

维护者尽力在**默认分支 / 当前开发版本**上修复安全问题；暂无长期维护的「仅安全修复」旧版本分支时，请以仓库内最新提交为准。

## 报告漏洞

若你发现 **pddbot** 中存在的可利用安全问题（例如凭据泄露路径、远程命令执行、可导致商家账户风险的缺陷），请 **不要** 在公开 Issue 中张贴完整利用细节或敏感数据。

建议流程：

1. 使用 GitHub **Security Advisories**（若仓库已启用）：**Security → Report a vulnerability**
2. 若不可用上述方式：向维护者发送 **私密邮件**至 **[zhaoyta@gmail.com](mailto:zhaoyta@gmail.com)**（主题建议包含 `security` 或 `pddbot`，便于分拣）。请勿在公开 Issue 中张贴漏洞细节；若只能通过 Issue，请仅索要「安全报告联系方式」，勿贴利用步骤。

请在报告中尽量包含：

- 受影响版本或提交范围、运行环境（OS / Python）
- 复现步骤（最小化）
- 潜在影响说明（不写攻击性payload亦可）

维护者会在合理时间内确认收到并协调修复与披露节奏；对善意披露者表示感谢。

## 范围说明

以下一般 **不属于** 本仓库安全响应范围（仍可通过普通 Issue 讨论）：

- 拼多多平台自身风控、封号策略
- 依赖第三方页面 DOM 变更导致的功能失效（除非你认为是本项目引入的不安全默认配置）

## 合规提醒

本工具涉及商家后台自动化，使用者需自行遵守平台规则与当地法律；安全修复不改变「合规使用」责任主体。
