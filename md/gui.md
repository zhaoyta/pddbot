# 桌面 GUI 设计

> 本文档记录 pddbot 桌面应用（PySide6）的 UI 结构、字段和数据流。

---

## 1. 启动方式

```
用户双击 start.sh / start.bat (终端运行也一样)
       │
       ├─[1] 检查 uv,缺失则用官方一行脚本装 (curl … | sh)
       ├─[2] uv venv --python 3.11      (uv 自带 CPython,不依赖系统 Python)
       ├─[3] uv pip install -r requirements.txt   (清华镜像由 uv.toml 固化)
       ├─[4] uv run python -m playwright install chromium
       └─[5] uv run python -m gui.app   ← 启动 GUI
```

每一步都幂等:已经装好的不会重装,只检查/补缺。
不需要先装 Python——`uv` 自己会拉对应版本的 CPython。

## 3. 各页面详细设计

### 3.1 首页（Home）

| 控件 | 说明 |
|---|---|
| 大启动按钮 | 文字「启动机器人」/「停止机器人」，颜色随状态切换 |
| 登录方式 | 复用登录态 / 重新扫码 |
| 保存 / 清除登录态 | 写回或删除 `storage_state.json` |
| 设置入口 | 文案链接跳转「设置」页（Bot 总开关、干跑、唤醒方式） |
| 状态卡片 | 今日回复 / 转人工 / 活跃客户 / 浏览器状态 |
| 状态栏补充 | 运行中状态文案会附带是否 DRY_RUN（值在「设置」页） |

**点击启动**：检查必要配置（API Key、商品映射等）→ 启 `BotWorker`；登录态路径以「设置」页或默认 `storage_state.json` 为准。
**点击停止**：发停止信号给 worker → 关 Playwright → 切回已停止。

### 3.2 设置（Settings）

| 控件 / 项 | 说明 |
|---|---|
| BOT_ENABLED | 总开关，关闭后只监听不自动回复 |
| DRY_RUN | 干跑：走 LLM 与 action_log，聊天框不真发 |
| 新消息来源 | 仅 HTTP（`/plateau/sync/message`、`chat/list` 等），说明见页面提示 |
| 登录态 JSON | `app.storage_state_path`：相对项目根或绝对路径，留空用默认 |
| 抓包目录 | `app.captures_dir`：探查脚本 `http_*.jsonl` / `chat_*.jsonl` 等输出目录 |
| 保存路径 | 按钮：将上述路径写入库（Bot 开关等为勾选即时写入） |
| 打开项目目录 | 系统文件管理器打开项目根 |
| 快捷按钮 | 跳转「应用」「风控」「模型」页 |

写入 SQLite `settings` 表；进入本页或 Bot 状态变化时会从库中再同步控件显示。

### 3.3 商品（Catalog）

商品 ↔ 资料映射 表格 CRUD。

| 列 | 说明 |
|---|---|
| 类型 | sku/goods/keyword 三种映射方式 |
| 主键 | skuId / goodsId / 关键字（逗号分隔） |
| 标题 | "S022-散打" 这种网盘文件名 |
| URL | 百度网盘分享链接（不含 pwd） |
| 提取码 | 4 位 |
| 操作 | 编辑 / 删除 / 测试预览 |

底部「新增 / 编辑 / 删除 / 命中测试」按钮。

数据存在 `db/pddbot.db` 的 `catalog_item` 表,`tools/catalog.py` 直接从这里查。

### 3.4 模型（Model）

| 字段 | 说明 |
|---|---|
| 提供方 | 下拉，前期只有 "DeepSeek" |
| Base URL | 默认 `https://api.deepseek.com` |
| API Key | 密码框（可显示） |
| 模型名 | 默认 `deepseek-chat`，下拉 + 自定义 |
| 温度 | 滑块 0.0 ~ 1.0，默认 0.3 |
| Max Tokens | 数字框，默认 800 |
| 测试连接按钮 | 调一次 `chat.completions.create` 看响应 |

保存写到 `settings` 表 key=`llm_*`，运行时 `llm/client.py` 优先从 `settings` 读。

### 3.5 阶段（StageReply）

5 个 stage（S0/S1/S2/S3/S4）每个一个折叠卡片：

```
┌─ S0 首次打招呼 ─────────────────────────┐
│  模式: ⚪ 智能（LLM）  ⚫ 自定义模板      │
│  ┌────────────────────────────────────┐ │
│  │ 您好,请问有什么可以帮您?            │ │
│  └────────────────────────────────────┘ │
│  [测试预览]   [恢复默认]                 │
└─────────────────────────────────────────┘
```

- **智能模式**：走当前 LLM Agent 逻辑
- **自定义模板**：固定文本，支持占位符 `{order_sn}` / `{goods_name}` / `{title}` / `{url_with_pwd}` 等
- **测试预览**：用 mock 上下文渲染模板，弹窗展示

数据写到 `stage_config` 表（已设计在 store.py）。

### 3.6 飞书（Notify）

| 字段 | 说明 |
|---|---|
| 飞书 webhook URL | 必填 |
| 启用通知 | 复选 |
| 通知事件 | 多选：转人工 / 核销失败 / 登录失效 / 每日报表 |
| 测试发送按钮 | 发一条测试消息 |

数据写到 `settings` 表 key=`notify_*`。

### 3.7 应用（AppConfig）

| 分组 | 字段（settings key） | 说明 |
|---|---|---|
| URL | `app.chat_url` / `app.login_url` / `app.redeem_page_url` / `app.order_list_api` | 留空则用 `core/config.py` 内置默认 |
| Bot | `bot.whitelist_uids` | 非空时仅处理列表中的买家 uid |

路径与抓包目录在「设置」页配置。

### 3.8 会话（Conversations）

```
┌──────────────────┬───────────────────────────────────┐
│ 客户列表(uid)     │  会话详情                          │
│ ┌──────────────┐ │  ┌──────────────────────────────┐  │
│ │ 14445...888  │ │  │ [客户] 你好             10:01 │  │
│ │ 14523...123  │ │  │ [Bot ] 您好,请问...      10:01 │  │
│ │ ...          │ │  │ [客户] 104396161488     10:02 │  │
│ └──────────────┘ │  │ [Bot ] 核销成功...      10:02 │  │
│                  │  │ [Bot ] 通过百度网盘...  10:02 │  │
│ 搜索: [_______]  │  └──────────────────────────────┘  │
│                  │  当前 Stage: S4 已发资料           │
└──────────────────┴───────────────────────────────────┘
```

- 左侧 QListWidget：所有有过对话的 uid（按最后活跃时间倒序）
- 右侧 QListWidget：消息流，区分客户/客服/bot 三种角色
- 数据来源：`chat_message` 表
- 顶部搜索框按 uid / 内容过滤

### 3.9 日志（Logs）

- QPlainTextEdit（只读、自动滚动）
- 实时显示 loguru 输出（通过 Qt Signal 推送）
- 顶部下拉过滤等级（DEBUG/INFO/WARN/ERROR）
- "打开 logs/ 目录" 按钮

## 4. 数据流

```
GUI 操作    ───┐
              │  写
              ▼
        ┌─────────────┐
        │   SQLite    │ ← Store (core/store.py)
        │             │
        │  settings   │
        │  stage_cfg  │
        │  chat_msg   │
        │  conv_state │
        │  order_state│
        │  card_code  │
        │  action_log │
        └─────────────┘
              ▲
              │  读
BotWorker  ───┘
(QThread)
   │ Playwright
   ▼
拼多多商家后台
```

- 所有运行时配置由 `core/settings.py` 提供，运行时**只读 settings 表，不读 .env**（.env 仅作首次默认）
- BotWorker 拿到的所有配置经 `core/settings.py`，所以 GUI 改完不用重启 bot 也能生效（除浏览器实例需要重启）

## 5. Bot 生命周期

```
[已停止] ──点启动──▶ [启动中] ──ready──▶ [运行中]
   ▲                                       │
   │                                       │
   └──── [关闭中] ◀──点停止 / 异常 ─────────┘
```

每个状态都通过 Qt Signal 广播到 GUI 各组件（首页大按钮、状态栏、菜单等）。

---

## 6. 不会做的事（边界）

- 不做用户登录系统（单机自用）
- 不做权限分级（一个人用，全开）
- 不做云端同步（数据全本地）
- 不做插件机制（先把核心做扎实）
- iOS / Android 不做，桌面三大平台够用
